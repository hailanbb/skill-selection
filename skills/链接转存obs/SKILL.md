# 链接转存 Obsidian 技能指南 (Save Link to Obsidian)

本技能旨在帮助用户实现高品质的网页技术文章永久归档。它支持环境自检与引导、首次运行时交互询问并保存用户的本地 Obsidian 库路径及图片存放偏好。

---

## 🛠️ 第一阶段：环境自检与首次初始化引导

当本技能被触发时，AI Agent 必须首先确保运行所需的基础环境和第三方依赖库已就绪：

### 1. 运行依赖项与可选 MCP/Skill 自检 (Doctor Check)
首先执行以下命令以验证 Python 基础环境和依赖包：
```bash
python "C:/Users/HiWin11/.gemini/config/skills/链接转存obs/scripts/save_to_obsidian.py" doctor
```

* **如果状态返回 `missing_dependencies`**：
  说明当前系统缺少核心解析库。AI Agent **必须直接执行命令**，自动帮用户安装：
  ```bash
  pip install beautifulsoup4 markdownify lxml
  ```
  安装成功后再次运行 `doctor`。

* **如果自检中的 `optional_mcp_status` 检测到可选组件/技能缺失**：
  AI Agent 应当在聊天中**主动引导用户**完善以下附加的 MCP 服务和 Skill 技能：
  
  1. **Enquire-MCP (本地混合语义检索)**：
     * *如果 `enquire_mcp_installed` 为 `false`*：AI 应当主动告知用户该组件可提供本地向量和全文检索。如果系统已安装 Node.js/NPM，AI 应当提议在终端中运行：
       ```bash
       npm install -g @oomkapwn/enquire-mcp
       ```
     * *如果 `enquire_mcp_registered` 为 `false`*：说明已全局安装但未在主客户端注册，AI 应向用户提供配置说明，或主动帮助用户在 `mcp_config.json` 中配置该服务器（配置格式参考 `serve` 命令与本地 `vault_path` 绑定）。
  2. **AnySearch (网页数据提取与搜索技能)**：
     * *如果 `anysearch_skill_installed` 为 `false`*：说明用户尚未安装 AnySearch 智能搜索提取技能（这正是本技能在直连防爬失效时的第一备选降级方案）。AI 应当在对话中主动提示用户：
       > “我检测到您尚未在本地安装 **AnySearch** 网页数据提取技能。AnySearch 是一个能够利用 14 个全球动态代理池执行网页抓取与搜索的技能，能作为微信直连防爬失效时的强力降级源。建议您使用 `/install anysearch` 命令或让我帮您将其安装到您的技能库目录下。”
  3. **Firecrawl-MCP (强力网页防爬与渲染爬虫)**：
     * *如果 `firecrawl_mcp_registered` 为 `false`*：AI 应当温和向用户说明这能大幅度提高微信公众号等复杂页面的抓取成功率。若用户拥有 Firecrawl 的 API 密钥（可到 [Firecrawl 官网](https://www.firecrawl.dev) 免费获取），AI 应当主动提议帮其将密钥配置进本机的 `mcp_config.json` 中。

### 2. 交互式库路径与图片偏好配置 (Onboarding)
在第一次运行抓取前，Python 脚本会检查全局配置。
* **如果执行 `fetch` 返回 `"status": "needs_config"`**：
  说明这是该用户首次使用本技能。AI Agent **必须暂停执行**，直接在聊天窗口中向用户提出两个澄清问题以获取本地路径：
  
  > 1. **您的 Obsidian 库（主库）在本地的绝对物理路径是什么？** (例如：`D:/obsidian/hailan`)
  > 2. **您希望将抓取的图片存放在主库根目录下的哪个纯中文目录名中？** (例如：`图片附件` 或者是 `附件`。我们将在此目录下为每篇文章建立独立的图片子文件夹，防止图片文件弄乱您的笔记目录。)

* **保存配置**：
  获取到用户输入的 `[库路径]` 和 `[图片目录名]` 后，AI Agent 必须执行以下命令将设置持久化保存到技能根目录下的 `config.json` 中：
  ```bash
  python "C:/Users/HiWin11/.gemini/config/skills/链接转存obs/scripts/save_to_obsidian.py" configure --vault "[库路径]" --image_dir "[图片目录名]"
  ```
  保存成功后，再继续执行后续的抓取工作。

## 🚀 第二阶段：核心执行工作流 (Workflow)

当初始化和配置就绪后，按照以下 5 个步骤执行归档：

### 步骤 1：智能多路抓取与图片本地化 (优先度设计)
为了保护用户本地 IP 不被微信防爬系统污染封锁，本技能在数据抓取时**必须严格遵循以下优先级层级**：

#### 🚀 优先级 1：优先采用云端防爬工具提取 (AnySearch / Firecrawl-MCP)
1. **优先调用 AnySearch 技能**：AI Agent 应当优先调用已安装的 `anysearch` 技能的 `extract` 命令（使用其 14 个全球动态代理）获取文章的 HTML 正文与原始标题/作者信息。
2. **备选调用 Firecrawl-MCP**：若 AnySearch 失败或未安装，且系统配置了 `firecrawl-mcp`，则调用 Firecrawl 的 scrape 接口抓取文章。
3. **执行图片本地化下载**：
   通过上述云端提取工具获取到正文 HTML 格式内容后，AI Agent **必须将 HTML 内容保存至会话的临时文件**中，并在终端执行以下命令进行本地主库图片下载和双链改写：
   ```bash
   python "C:/Users/HiWin11/.gemini/config/skills/链接转存obs/scripts/save_to_obsidian.py" localize --title "<文章标题>" --html "<临时HTML文件路径>"
   ```
   *这会自适应读取 `config.json` 并把图片物理下载到 `<库路径>/<图片目录名>/<文章标题>/` 目录下，并返回重构了 `![[<图片目录名>/文章标题/image_x.png]]` 的净化 HTML 内容。*

#### 🛡️ 优先级 2：本地 Python 伪装直连抓取 (最终兜底 Fallback)
若上述优先级 1 的云端防爬途径未安装或运行报错，AI Agent 将**退而求其次在本地执行 Python 直连抓取**。
该脚本会自动伪装真实的 Chrome User-Agent 报头和 Referer（`mp.weixin.qq.com`）直接穿透抓取：
```bash
python "C:/Users/HiWin11/.gemini/config/skills/链接转存obs/scripts/save_to_obsidian.py" fetch --url "<用户提供的链接>"
```
*该脚本会自动完成 fetch、图片本地化下载和链接重构，直接输出含有 Obsidian 相对双链的 HTML。*

#### ⚠️ 优先级 3：人工复制输入兜底
若本地直抓依然报错（强人机滑块拦截）：
1. 引导用户“在浏览器中打开链接，全选并复制正文文本发给我”。
2. 拿到正文后，直接进入 **步骤 2**。

---

### 步骤 2：广告与软文深度净化规则 (AI 语义过滤)
获取到正文内容后，**AI Agent 必须调用自身的语言模型理解力，严格按照以下净化准则对文本进行过滤净化**，剔除所有“噪点”：

#### 1. 广告识别能力
* **硬广告识别**：精准识别文章中直接的商业宣传、产品推销、促销信息等。
* **软文甄别**：辨别隐藏在文章内容中的软性推广、品牌植入、产品推荐等。
* **营销话术识别**：识别 “限时优惠”、“立即购买”、“点击链接”、“扫码关注” 等引导性用语。
* **推广链接检测**：自动发现并移除文章中夹带的推广超链接、微信号、二维码图片、联系方式等推广元素。

#### 2. 内容处理能力
* **精准删除**：在去除广告内容的同时，确保不误删正文的核心知识信息。
* **语义连贯修复**：删除推广段落后，对上下文进行自然过渡处理，保持文章流畅性。
* **格式优化**：清理因删除广告而产生的多余空行、断裂段落等格式问题。
* **信息忠实保留**：严格保留原文的客观事实、观点表达和知识内容，不添加不删减。

#### 3. 判断分析能力
* **产品测评甄别**：区分客观的产品使用体验与商业推广软文。
* **正常引用的识别**：区分合理的品牌提及案例与刻意营销植入。
* **行业术语判断**：准确区分行业通用术语与特定品牌推广用语。
* **公益宣传识别**：识别政府公告、公益广告等非商业性推广，避免误删。

#### 4. 核心净化原则
* **广告清零**：凡是具有商业推广性质的内容，无论呈现形式如何，一律清除。
* **内容保真**：坚决保护原文的非商业信息，包括但不限于知识科普、新闻事实、观点评论。
* **格式整洁**：清理后的文章应保持完整的结构和良好的阅读体验。
* **最小干预**：仅删除广告内容，不对原文进行主观改写或增补。

#### 5. 具体处理准则
* **推广链接一律移除**：包括但不限于商品链接、二维码、微信号、推广短链接。
* **营销话术全面清除**：如 “加微信”、“点关注”、“领资料”、“限时”、“错过再等一年” 等。
* **软文段落整体删除**：以推广为目的的完整段落应整段移除。
* **品牌名称灵活处理**：如为产品测评中的客观提及可保留，如为推广软文则清除相关段落。

#### 6. 严格禁止事项
* **禁止修改原文事实**：不添加、不篡改、不歪曲原文表述的任何事实信息。
* **禁止过度净化**：不将正常的产品讨论、行业分析误判为广告而删除。
* **禁止夹带私货**：不在净化过程中植入自己的观点或第三方推广内容。
* **禁止破坏结构**：删除广告后必须修复段落，避免出现断裂、重复等内容问题。

---

### 步骤 3：智能生成纯中文分类名
根据净化后的正文核心主题，由 AI Agent 智能判定一个**最合适、最简练的纯中文分类名**（禁止产生任何英文或拼音目录），例如：`AI智能体`、`知识管理`、`编程开发` 等。

---

### 步骤 4：排版美化与收件箱写入
1. 将净化后的正文整理为排版精美的 Markdown 格式。
2. **排版美化准则**：
   * 合并微信排版中因适配手机屏幕产生的过多碎片换行，段落保持紧凑。
   * 使用 Obsidian 风格的 Callout 块（如 `> [!tip]`、`> [!abstract]`）对核心摘要、开源地址或重要提示进行视觉包裹。
   * 使用 `==重点高亮==` 标注文章里的硬核概念，代码块必须配置对应的语法高亮。
3. **元数据 Frontmatter 配置 (中文化)**：
   必须在 Markdown 笔记顶部配置以下结构化 YAML 属性：
   ```yaml
   ---
   title: "文章标题"
   author: "作者名称"
   source: "文章原始 URL 链接"
   date: YYYY-MM-DD
   category: "待阅收集箱 / <步骤3确定的分类名>"
   tags:
     - 待阅/收集
     - 技术/<对应的二级中文领域，如人工智能/知识库>
   summary: "AI 提炼的一句话核心内容摘要"
   ---
   ```
4. **保存路径**：
   将最终的 Markdown 笔记写入至库根目录下的 **“待阅收件箱”** 中，格式为：
   `[库路径]/待阅收件箱/<分类名>/<清理特殊字符后的中文文章标题>.md`

---

### 步骤 5：自动同步更新检索索引
文件保存成功后，立即在终端执行同步索引命令，使文章即刻在本地可被 `enquire-mcp` 混合检索：
```bash
python "C:/Users/HiWin11/.gemini/config/skills/链接转存obs/scripts/save_to_obsidian.py" index
```

当上述工作完成后，AI Agent 应当向用户做一份简洁直观的结项汇报，必须包含：
1. **抓取所用通道**：明示本次是通过 AnySearch、Firecrawl 或是本地直连兜底抓取成功。
2. **抓取结果与净化详情**：简述抓取的文章标题、作者、以及根据广告过滤准则清除了哪些推广广告与软文。
3. **分类与归类目录**：标明智能分配的**纯中文分类名**，并列出笔记在 Obsidian 中的 Clickable 绝对路径链接。
4. **图片本地化统计**：说明已将几张图片安全下载至全局根目录下的 `<图片目录名>` 文件夹中。
5. **索引同步成功通知**：知会用户 `enquire-mcp` 的全文检索与向量数据库均已重新索引同步完毕。
