# 洁癖收尾与规范审计技能使用说明 (Neat-Freak Guide)

本技能专为 Antigravity 客户端及 Windows 开发环境定制，旨在会话结束前对当前项目的文档（`README.md`、`docs/`）、规则文件（`AGENTS.md`）与大脑临时资产进行深度审查、自动毕业收缩，并强力核验本地路径 Clicklink 规范、WPS 自动化格式及前端美学。

---

## 🛠️ 第一阶段：首次初始化配置与自检 (Onboarding & Doctor)

当本技能被触发时，AI 代理会检测当前项目是否已做好规范审计的准备，并执行自愈式初始化。

### 1. 运行自测与自动初始化 (Onboarding)
在您对 AI 代理发出“收尾”或“整理”指令后，AI 会自动探测当前工作空间根目录下是否存在 `.agents/AGENTS.md` 规则手册。
* **如果规则文件不存在**，AI 代理会在聊天中进行如下交互询问：
  > **未在当前项目发现 `.agents/AGENTS.md` 规则配置文件。是否需要为您自动生成规范脚手架？**
* **自动生成**：若您回复同意，AI 会在当前工作空间自动创建 `.agents/` 目录并生成默认的 `AGENTS.md`，自动将全局规则库中的重要硬边界红线与 Clicklink 指针表作为初始配置写入。

### 2. 本地自检与环境评估
AI 代理在执行审计前，会评估当前项目的文档健康度：
- 检查项目文档总体量与脑区临时资产的体积比。若临时脚本或 walkthrough 行数远大于项目 `docs/`（体量倒挂），AI 将在控制台输出警告，提示需进行高优先级“毕业”清理。

---

## 🚀 第二阶段：核心执行工作流 (Workflow)

当自检完成后，AI 代理会按照“洁癖级”收尾清理逻辑执行以下工作流：

### 步骤 1：盘点现状 (Mechanic Enumeration)
AI 会列出当前项目的所有 Markdown 文件（包括 `README.md`、`docs/*.md`、`AGENTS.md`），以及当前会话脑区中的所有文件。生成一份评估变更的待办清单，确保没有漏掉任何文档。

### 步骤 2：规范执行审计 (Compliance Audit)
AI 会拿着本技能配置的 [governance.md](references/governance.md) 校验当前项目代码及文档：
1. **Windows 路径 Clicklink 审计**：检查并修复所有使用反斜杠或裸路径书写的本地绝对与相对路径，自动转换为 `[basename](file:///path/to/file)` 格式的 Clickable links。
2. **WPS 交付格式审计**：若有生成的 Excel 报表，审查表头是否为冷色调加粗背景、数值是否格式化、列宽是否自适应、是否开启了 AutoFilter 筛选。
3. **Web 美学与沙盒审计**：审计前端页面是否采用高品质 CSS，清除临时沙盒（`scratch/`）之外所有硬编码的历史会话 UUID 路径。

### 步骤 3：识别与同步变更 (Identify Changes)
对比本次会话的 Git 改动与核心逻辑变更，根据 [sync-matrix.md](references/sync-matrix.md) 识别受波及的文档层级：
- API/路由增加 → 同步改动项目级 `AGENTS.md` 路由表与 `docs/integration-guide.md`。
- 符号/功能退役 → 运行 `grep` 清理所有文档中的死引用与失效代码示例。

### 步骤 4：物理修改与脑区清理 (Cleanup)
- **实际编辑**：使用文件修改工具将改动真正写入文档中，坚持“减优于加”、“合并优于追加”的编辑原则。
- **物理清理**：彻底物理删除脑区 `scratch/` 目录下调试留下的一次性临时脚本，保持会话环境绝对纯净。

### 步骤 5：清单自检与摘要输出 (Checklist & Summary)
- AI 逐项对照 Checklist 完成自检，并在对话框中以行动导向模版输出一份结构清晰的 **《同步与审计完成》** 摘要汇报，明确区分“自动修复项”与“待您拍板确认项”（如物理删除文件、处理规则冲突等）。
