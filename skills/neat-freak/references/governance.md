# 治理与审计规范 (Governance)

本规范为 `neat-freak` 技能在执行“规范执行审计（规则 → 实践）”时的核心判据、处置分级与自检标准。作为 Antigravity 的专属审计指南，本规范必须被严格、机械地执行，以保证项目质量和注意力资产的整洁。

---

## 1. 核心审计判据

### 1.1 Windows 本地路径 Clicklink 规范
在 Windows 运行环境下，为了保证用户在 IDE 界面中可以直接点击跳转，必须对文档和规则中出现的所有本地路径执行如下校验：
- **正斜杠标准化**：路径中所有的反斜杠（`\`）必须转换为正斜杠（`/`）。
- **Clickable 链接包裹**：所有本地绝对路径或相对路径，必须使用 markdown 格式链接进行包裹，且其链接地址必须使用 `file:///` 协议。
  - *正确示例*：`请查看 [runbook.md](file:///D:/project/docs/runbook.md)`
  - *错误示例*：`请查看 D:\project\docs\runbook.md`
- **行范围指向**：对于指向具体代码行范围的链接，使用 `file:///path/to/file#L123-L145` 格式，且链接文本禁止被包裹在反引号（\`）中。
  - *正确示例*：`参考 [main.py:L10-25](file:///D:/project/main.py#L10-L25)`
  - *错误示例*：`参考 [\`main.py\`](file:///D:/project/main.py#L10-L25)`

### 1.2 WPS 自动化交付规范
若项目包含向 WPS Office（Word / Excel / PPT）输出自动生成报告或处理数据的逻辑，必须审计所编写的脚本和输出模版是否符合以下标准：
- **数据表格 (.xlsx)**：
  - **表头设计**：表头必须加粗，并配置专业冷色调背景（如深蓝、灰黑，禁止使用刺眼的高饱和度暖色调）。
  - **数值格式化**：金额、数量、百分比等数值类数据必须配置显式的数据格式化，严禁呈现为裸数字。
  - **排版安全**：必须配置列宽自适应，防止出现文字被截断或显示为 `###` 的情况。
  - **筛选配置**：默认必须对数据列开启自动筛选（AutoFilter）。
- **文字文档 (.docx)**：
  - 必须严格保持 H1, H2, H3 的清晰标题层级。
  - 排版行间距需符合专业阅读习惯，以提供极佳的可读性（Scannability）。
- **演示文稿 (.pptx)**：
  - 整体风格必须克制、严密，具有商业高级感。
  - 严禁使用过度花哨的装饰、动画或卡通化元素。

### 1.3 Web UI 美学与动态沙盒规范
若项目涉及 HTML、CSS 或前端网页组件设计，必须审计：
- **视觉品质**：是否使用了精心调配的渐变色和现代无衬线字体。杜绝简陋、刺眼的纯色或未修饰的默认浏览器样式。
- **不可使用 Placeholder**：任何地方不得残留临时的 Placeholder 占位图。如需图片展示，必须使用 `generate_image` 工具生成真实的高品质素材并保存在本地项目中。
- **动态沙盒原则**：
  - 代码和文档中严禁硬编码任何历史会话的 UUID 路径。
  - 临时生成用于 debug 的测试脚本、缓存或中间数据，必须存放在当前会话的 `scratch/` 目录中。

## 2. 处置分级

审计中发现遇到的违规行为或问题，必须按如下分级进行处置：

### 2.1 自动修复类（直接操作）
对项目和协作环境无破坏性、纯补齐或格式化的修改，应由 AI 直接完成修改，并在最终的变更摘要中向用户汇报。
- **范围**：
  - 修复未标准化的 Windows 路径（将反斜杠替换为正斜杠，包裹 `file:///` 链接）。
  - 修复 `.gitignore` 中遗漏的敏感配置（如 `.env` 等）。
  - 修复缺失的 `AGENTS.md` 软链接。
  - WPS 或 Web 资源中，非逻辑性、纯格式化的安全修改。

### 2.2 待你拍板类（只收集，不上手）
具有破坏性风险、涉及多方决策、可能破坏他人引用或部署脚本的改动，AI 必须保持克制，不可自行修改。需在变更摘要中专设「待你拍板」板块并附上修改方案和代价分析。
- **范围**：
  - 目录重命名或文件删除。
  - 合并两个有冲突的、内容不同的项目规则文件。
  - 清理无法确认是否已完全退役的死引用规则。
  - 修改核心系统的环境变量默认值。

## 3. 会话收尾自检清单

在执行收尾前，必须机械地逐项核对并打勾：

- [ ] **路径标准化**：项目中所有 markdown 文件和规则文件里的 Windows 本地路径均已转换成了 clickable links（`file:///` 格式且为正斜杠）。
- [ ] **路径无反引号**：链接文本中没有错误包裹反引号（形如 `[`file`](...)` 已被修正为 `[file](...)`）。
- [ ] **规则层去历史叙事**：`AGENTS.md` 中没有新增历史性流水账（"X 功能已上线"之类的内容已迁移至 `docs/CHANGES.md`）。
- [ ] **记忆与大脑毕业**：在 `task.md` 或 `walkthrough.md` 中浮现的项目常驻原则和踩坑教训已并入项目文档或 `AGENTS.md`，脑区中无用的临时 debug 脚本已清空。
- [ ] **WPS 规范达标**：脚本生成的 xlsx 具备冷色调加粗表头、列宽自适应、自动筛选及格式化数值。
- [ ] **Web 界面质感**：HTML/CSS 界面不含 Placeholder，样式具有 Premium Aesthetics。
- [ ] **无硬编码 UUID 路径**：排查并清除了所有残留的旧会话脑区 UUID 路径，临时文件已妥善归档。

## 4. 变更摘要模板

```markdown
## 同步与审计完成

### 大脑与记忆毕业
- **毕业项目**：xxx（并入 [docs/xxx.md](file:///path/to/docs/xxx.md)）
- **清理项**：清空了脑区 `scratch/` 中的临时 debug 脚本。

### 文档与规则变更
- **[项目名]/AGENTS.md** — 更新了环境变量清单与 Clicklink 路径规范。
- **[项目名]/docs/CHANGES.md** — 增加了本会话的特性发布与退役历史记录。

### 规范审计结果
- **自动修复**：
  - 将 [README.md](file:///path/to/README.md) 中 3 处反斜杠路径标准化为了 Clickable Markdown Links。
- **待你拍板**：
  - [ ] 发现 `AGENTS.md` 中引用了已删除的脚本路径 `C:/old_scripts/test.py`。建议清除该条规则（置信度：高）。
  - [ ] 检测到同类违规重复出现：连续 3 次发现 `.env` 规则未被遵守。建议将此项规则转为 Git pre-commit hook 进行强制拦截。

### 规范体检指标
- **[项目名]/AGENTS.md**：120 行 / 6KB —— 状态健康。
```
