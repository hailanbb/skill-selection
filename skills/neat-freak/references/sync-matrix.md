# 变更影响矩阵 (Sync Matrix)

本矩阵定义了在 Antigravity 开发过程中，各种技术变更应当波及并同步更新的文档和规则层级。在执行 `neat-freak` 任务时，必须依据此矩阵进行双向核验，防止文档与代码发生漂移。

## 变更同步映射表

| 变更类型 | 影响的代码资产 | 必须同步的规则文件 (`AGENTS.md`) | 必须同步的文档 (`docs/` 与其他) |
| :--- | :--- | :--- | :--- |
| **API 接口 / 路由** | 新增、删除或修改路由控制器（如 FastAPI, Express, Spring 等） | **项目级 `AGENTS.md`**：<br>更新路由速查表、接口权限模型及红线。 | **`docs/integration-guide.md`**：<br>更新接口文档、错误码定义及 curl 测试示例。<br>**`docs/architecture.md`**：<br>更新系统数据流与接口架构图。 |
| **环境变量** | 新增、修改或废弃配置类、配置读取逻辑、`.env` 文件 | **项目级 `AGENTS.md`**：<br>更新环境变量说明表（说明键名、用途、默认值及是否必填）。 | **`docs/runbook.md`**：<br>更新本地与线上部署的启动配置说明。 |
| **数据库模型 (Schema)** | 新增表/集合、修改字段、新增索引或关系映射 | **项目级 `AGENTS.md`**：<br>更新核心表与敏感字段的读写约束规则。 | **`docs/architecture.md`**：<br>更新数据模型（Data Model）章节、ER 图及字段类型说明。 |
| **WPS 自动化产物** | 修改了生成 Word、Excel、PPT 办公文档的 Python/VBA/JS 脚本逻辑 | **项目级 `AGENTS.md`**：<br>更新 WPS 自动化排版约束与审计红线（冷色调、自适应、表头加粗等）。 | **`docs/runbook.md`**（或交付说明）：<br>更新交付标准、冒烟测试验证指标。 |
| **退役 / 改名 / 下线** | 废弃特定功能模块、下线接口、删除废弃脚本或子模块 | **项目级及全局 `AGENTS.md`**：<br>清理废弃指令、环境变量及死引用路径。 | **`docs/CHANGES.md`**：<br>记录功能退役事件。<br>**全体 `docs/`**：<br>全局 `grep` 被删符号，清理废弃的示例代码与引用。<br>**脑区 `scratch/`**：<br>彻底清除本地无用的旧测试脚本。 |
| **跨项目依赖变更** | 修改了公共 SDK、依赖包版本、或者上游服务的 API 契约 | **两个项目的 `AGENTS.md`**：<br>更新依赖版本边界规则与通信权限。 | **上下游的 `docs/`**：<br>同步修改上游的接口定义和下游的接入指南，确保两端对齐。 |

---

## 关键同步审计原则

1. **下线双清原则**
   当代码中某个 API 或环境变量退役时，除了删除代码，**必须在同一次同步中清空 `docs/` 和 `AGENTS.md` 中的所有非历史记录性引用**。未被清干净的死符号是造成后续 Agent 逻辑误判的主要元凶。
   
2. **跨项目对齐原则**
   如果本次会话修改了项目 A（上游），并且项目 B（下游）依赖它，那么**项目 B 的 docs 也要在同一次会话里更新**。严禁仅更新上游而遗漏下游。

3. **受众隔离原则**
   - 规则文件 (`AGENTS.md`) 只保留对下一次开发有指导意义的约束和红线，不写详细的机制和使用说明。
   - 使用说明、架构和运维细节一律归于 `docs/`，并在 `AGENTS.md` 的「深入文档指针表」中用 Clicklink 指向具体文件。
