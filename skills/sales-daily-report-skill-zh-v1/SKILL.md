# 销售日报整理与优化 Skill

## 1. 适用场景与边界

- **适用场景**：当用户发送销售日报片段、要求查看今日日报进度、或需要合并整理优化日报并写入本地 [日报汇总.md](file:///d:/GoogleAI/work/日报汇总.md) 时。
- **限制与边界**：
  - 本技能不限制销售团队的具体人数及姓名，所有配置均读取自本地的 `d:/GoogleAI/work/日报管理/config.json`，确保代码与人名去隐私化。
  - 依赖本地 Python 状态管理器 `scripts/manage_reports.py` 进行状态的持久化维护。

## 2. 运行环境检查与初始化

首次在新的 PC 或新用户环境下使用此技能时，必须优先进行环境检查：
1. **环境检查**：运行 `python scripts/manage_reports.py --check-env`，确保 Python 3 可用、目录有读写权限且全局/本地技能库中已安装依赖的 `humanizer-zh` 技能。
2. **初始化配置**：如果发现配置文件 `d:/GoogleAI/work/日报管理/config.json` 不存在，运行 `python scripts/manage_reports.py --init` 创建一个脱敏的默认配置模板。
3. **团队配置确认**：引导用户设置销售人员名单、顺序、高亮人名颜色代码（默认 `#1a73e8`）及是否仅输出“复制版”（默认双版本）。

## 3. 人员与配置管理指令

Agent 可随时协助用户通过命令行管理团队成员变动及全局设置：
- **新增销售人员**：`python scripts/manage_reports.py --add-reporter --key <英文标识Key> --fullname <中文展示名> --aliases <逗号分隔别名列表>`
- **更改成员状态 (活跃/请假/离职)**：`python scripts/manage_reports.py --set-status --key <Key> --rep-status <active|leave|inactive>`
  - `active`：活跃。正常参与每日进度统计。
  - `leave`：请假。今日进度统计自动排除此人，合并时自动填充为 `【请假】`。
  - `inactive`：离职/不再统计。系统完全忽略，且不显示此人。
- **调整全局配置 (高亮色 / 输出格式)**：`python scripts/manage_reports.py --config --name-color <HEX代码如#1a73e8> --output-format <both|copy_only>`

## 4. 工作流程与指令

### Step 1: 识别与存储
- 用户发送任何包含日报内容的文本时，首先匹配汇报人姓名或别名。
- 调用 `python scripts/manage_reports.py --save --content "<日报内容>"` 保存。
- **🔴 CHECKPOINT / 异常捕获**：
  - 如果脚本返回 `"success": false` 或无法自动识别汇报人，**必须**暂停并询问用户：“未能识别出该日报属于哪位成员，请问这是谁的日报？”
  - 获得确认后，重新以 `--name "<确认的Key>"` 保存。

### Step 2: 状态展示
- 每次成功保存日报后，读取状态。
- 向用户展示当前的**今日日报提交进度看板**，例如：
  > 📢 **今日日报收集进度 (4/6)**
  > - **已提交**：Alice（销售A）、Bob（销售B）、Charlie（销售C）
  > - **未提交**：David（销售D）、Eva（销售E）
  > - **请假**：Grace（销售F）
  > 
  > 💡 *已为您妥善保存已收到的日报。当检测到所有活跃成员全部提交后，将自动启动深度整理和优化。您也可以发送“强行整理日报”提前生成。*

### Step 3: 触发整理与优化
- **触发条件**：
  1. 状态显示所有活跃成员已全部提交。
  2. 或者用户明确发出“强行整理日报”指令。
- **数据获取**：调用 `python scripts/manage_reports.py --get-all` 读取所有内容。

### Step 4: 文本深度优化（核心大模型处理）
对拼接后的文本运行以下优化策略：
1. **繁体转简体**：将所有繁体字、粤语常用字转换成对应的规范中文简体字。
2. **去除表情杂符**：过滤掉所有 Emoji 表情和多余的无意义特殊符号、大段空白。
3. **AI 痕迹消除与人性化改写 (融合 humanizer-zh)**：
   - 按照 `humanizer-zh` 规范进行人性化脱敏，使表达更自然、通顺，杜绝八股味和机器生成感。
   - 过滤并替换高频 AI 词汇（如：‘此外’、‘然而’、‘至关重要’、‘深入探讨’、‘格局’、‘标志着/象征着持续的...’）。
   - 删除没有实际业务含义的通用积极结论和夸大性象征意义，转为直接陈述具体的业务事实。
   - 丰富句子节奏，长短交错，多使用具体细节和动词。
   - 列表项避免同义词循环与机械性加粗重复。
4. **逻辑与语法润色**：修正明显语病或逻辑生硬的句子，保持原意不变，禁止删减关键业务数据（金额、数量等）。
5. **钉钉文档复制排版优化**：
   - **人名高亮 (H2)**：成员名字作为二级标题，并高亮为配置的 `name_color` 色彩（默认 `#1a73e8`）：`## <font color="#1a73e8">Alice（销售A）</font>`。
   - **业务分类 (H3)**：大类作为三级标题：`### 一、 渠道商务拓展`。
   - **细分渠道/网点 (H4)**：具体拜访的诊所、大客户名称等作为四级标题：`#### 某某医疗中心`。
   - **工作细节 (单层列表 -)**：工作详情以单层无序列表项形式列出：`- **项目进度**：...`。
   - 每一层级排版中禁止使用多级缩进列表，确保粘贴进钉钉文档后，各类标题与列表能完美还原高矮胖瘦和不同字号。

### Step 5: 输出、归档与汇总
- 按照 `output_format` 配置，如果为 `both`，输出**直接可读的预览版**和 **Markdown 代码块**；如果为 `copy_only`，则只输出 **Markdown 代码块**。
- **🔴 CHECKPOINT / 自动汇总与清理**：
  - 调用 `python scripts/manage_reports.py --archive --content "<整理后的日报汇总内容>"`。
  - 该命令会自动将日报内容以倒序追加到本地工作区根目录的 [日报汇总.md](file:///d:/GoogleAI/work/日报汇总.md) 文件的顶部。
  - 追加并归档完成后，脚本将自动清理 `temp/` 目录下的所有临时文件。
  - 最终在页面末尾提示：“今日日报已成功追加并汇总至本地 [日报汇总.md](file:///d:/GoogleAI/work/日报汇总.md)，临时文件已自动清理干净。”

## 5. 操作反例黑名单（不要做的事）
- 🚫 **绝对不要保留任何繁体字**：翻译一定要彻底，包括港台习惯用语转换为内地常用词。
- 🚫 **绝对不要保留 Emoji 绘文字**：确保最终输出没有任何 📸, 🌟, 🚀 等表情，使文档严肃专业。
- 🚫 **绝对不要保留任何真实销售人员姓名在技能描述或代码库中**：必须做到去隐私化，仅保存在用户本地 `config.json` 中。
- 🚫 **绝对不要在未收齐时静默整理**：除非用户明确指令（“强行整理”），否则在未收到全部活跃成员日报前，仅更新状态，绝不提前合并。
- 🚫 **绝对不要随意变更汇报顺序**：汇报顺序必须严格按照 `config.json` 中 `reporters` 数组的物理顺序输出。
