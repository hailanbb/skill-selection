# 销售日报整理至钉钉技能使用说明 (Sales Daily Report for DingTalk)

本技能用于自动收集、整理和优化销售团队的日报，并将其排版为适合钉钉文档粘贴的格式。它支持环境自检与引导、首次运行时交互询问并保存本地配置，并且在合并完成后自动将日报追加到本地的 `日报汇总.md` 顶部，随后彻底清理所有临时文件。

---

## 🛠️ 第一阶段：环境自检与首次初始化引导

当本技能被触发时，AI Agent 必须首先确保运行所需的基础环境和依赖已就绪：

### 1. 运行依赖项与可选 Skill 自检 (Doctor Check)
首先在终端中执行以下命令以验证运行环境：
```bash
python scripts/manage_reports.py --check-env
```

* **如果自检中的 `humanizer_installed` 为 `false`**：
  说明当前系统缺少核心的人性化润色技能 **`humanizer-zh`**。AI Agent 应当在对话中**主动引导用户**：
  > “我检测到您尚未在本地安装 **humanizer-zh** 技能。该技能是实现日报自然通顺、去除 AI 八股味的核心依赖。建议您让我帮您将其安装到您的全局技能库（`%USERPROFILE%/.gemini/config/skills/`）中。”

---

### 2. 交互式团队与偏好配置 (Onboarding)
在第一次运行收集前，Python 脚本会检查本地配置。
* **如果执行 status 返回为空或缺少配置文件**：
  说明这是用户首次使用该技能。AI Agent **必须暂停执行**，直接在聊天窗口中向用户提出以下澄清问题以获取团队偏好配置：
  
  > 1. **请问您的销售团队成员名单是什么？** (请提供各位销售的中文姓名及用于匹配日报的别名/英文名，例如：`Terence（杨文皓）`, `Rubens（林秀柽）`)
  > 2. **请确认生成后的销售人员顺序。** (合并后的日报将严格按照此物理顺序排版输出)
  > 3. **人名在钉钉文档中高亮显示的 HEX 颜色代码是多少？** (默认使用海洋蓝：`#1a73e8`)
  > 4. **最终是否直接输出“复制版”代码块？** (默认输出“预览+复制版”，您也可以选择“仅直接输出复制版”以方便快速一键复制代码)

* **保存配置**：
  获取到用户输入的偏好后，AI Agent 必须执行以下命令将设置初始化并持久化保存到外部的 `config.json` 中：
  ```bash
  # 初始化配置
  python scripts/manage_reports.py --init
  
  # 动态配置人名颜色与输出模式
  python scripts/manage_reports.py --config --name-color "#1a73e8" --output-format "both"
  
  # 逐一添加销售人员
  python scripts/manage_reports.py --add-reporter --key "Sales_A" --fullname "Alice（销售A）" --aliases "alice,销售a"
  ```
  保存成功后，再继续执行后续的收集工作。

---

## 🚀 第二阶段：核心执行工作流 (Workflow)

当初始化和配置就绪后，按照以下 5 个步骤执行收集与整理：

### 步骤 1：零星保存日报与进度展示
* 用户发送销售人员的日报时，系统会自动识别人名。若无法识别，AI 会主动询问用户确认。
* 在终端执行保存：
  ```bash
  python scripts/manage_reports.py --save --content "<日报内容>"
  ```
* 每次保存成功后，读取当前进度状态，展示**今日日报提交进度看板**，清晰呈现：
  * 已提交人员名单
  * 未提交人员名单
  * 请假人员名单

---

### 步骤 2：文本深度优化（AI 语义过滤）
当所有活跃成员全部提报完成（或用户发出“强行整理日报”指令）后，调用 `python scripts/manage_reports.py --get-all` 读取所有内容，AI Agent 必须执行以下优化策略：
1. **繁体转简体**：将所有繁体字、粤语常用字转换成对应的规范中文简体字（例如：“對接”->“对接”，“跟進”->“跟进”）。
2. **去 Emoji 杂符**：过滤掉所有 Emoji 表情（如 🚀, 📸, 📢, 🔥 等）和多余的无意义特殊符号、大段空白。
3. **人性化脱敏与改写 (融合 humanizer-zh)**：
   - 按照 `humanizer-zh` 规范进行人性化脱敏，使表达更自然、通顺，杜绝八股味和机器生成感。
   - 替换高频 AI 词汇（如：‘此外’、‘然而’、‘至关重要’、‘深入探讨’、‘格局’、‘不仅...而且...’）。
   - 删除没有实际业务含义的通用积极结论和夸大性象征意义，转为直接陈述具体的业务事实。
   - 列表项避免同义词循环与机械性加粗重复。

---

### 步骤 3：钉钉文档复制排版优化
为了防止粘贴进钉钉文档时格式错乱，**必须**使用“纯标题层级结构”进行排版：
- **人名高亮 (H2)**：成员名字作为二级标题，并使用配置的高亮色高亮：`## <font color="#1a73e8">Alice（销售A）</font>`。
- **业务分类 (H3)**：大类（如一、二、三）作为三级标题：`### 一、 渠道商务拓展`。
- **细分网点 (H4)**：具体拜访的诊所、大客户名称等作为四级标题：`#### 某某医疗中心`。
- **工作细节 (单层列表 -)**：工作详情以单层无序列表项形式列出，禁止多级缩进：`- **项目进度**：...`。

---

### 步骤 4：自动汇总与临时文件清理
1. 输出优化后的日报全文（包含可读预览及 Markdown 复制代码块）。
2. 调用命令进行归档和本地汇总追加：
   ```bash
   python scripts/manage_reports.py --archive --content "<整理后的日报汇总内容>"
   ```
   * 脚本会自动将今日日报以倒序方式追加写入本地工作区根目录的 [日报汇总.md](file:///d:/GoogleAI/work/日报汇总.md) 文件顶部（最新的日期在最上面）。
   * 追加并归档完成后，脚本将自动彻底清理 `temp/` 目录下的所有临时文件，保证环境整洁。
